## 1. Goal and success criteria

### Goal

Prove the architecture works end-to-end using only free data. Build the foundation that will support paid data integration in v0.1.2 without requiring protocol changes. The deliverable is a Python package that replicates the Fama-French 5-factor model on daily-bar data, validated against both OSAP and Ken French published returns.

### Success criteria

v0.1 is complete when all of the following are true:

1. The package `alpha-decay-foundry` is installable via `pip install -e .` from the repository root.
2. Running `pytest` produces zero failures and ≥85% line coverage on `src/alpha_decay_foundry/core/`.
3. Running `python examples/replicate_ff5.py` reproduces the Fama-French 5-factor daily returns from 1963-2024 with tracking error within 5 bps standard deviation against Ken French's published daily returns, AND within 5 bps against OSAP's portfolio returns aggregated to monthly.
4. The `AsOfDataProvider` correctly raises `LookAheadError` when a strategy attempts to access future data, demonstrated by a test that includes an intentional look-ahead bug and confirms the framework catches it.
5. Information Coefficient and Information Ratio are computed and reported for every strategy.
6. The fundamental law decomposition (IR ≈ IC × √breadth) is computed and reported.
7. The DuckDB + Parquet caching layer correctly persists downloaded data and reloads from cache on subsequent runs.
8. The package builds successfully on GitHub Actions for Python 3.11 and 3.12.
9. README.md is complete with project description, installation, quickstart example, link to context document.
10. Type checking passes with `mypy src/alpha_decay_foundry --strict`.
11. CONTRIBUTING.md documents the issue-PR-audit workflow.

If any of these fail, v0.1 is not complete.

### Explicit non-goals

These are out of scope for v0.1 and should not be implemented:

- Any data source other than OSAP, Ken French, and a yfinance fallback for ad-hoc inspection
- Any paid data integration (deferred to v0.1.2)
- Real point-in-time index membership (use OSAP universe coverage in v0.1; real point-in-time in v0.1.2)
- Computing signals from raw fundamentals (use OSAP's pre-computed characteristic values in v0.1)
- zipline-reloaded engine adapter (v0.2)
- Portfolio optimizers beyond trivial weight assignment (v0.2)
- Risk overlays (v0.2)
- Slippage, transaction cost, and tax models (v0.2)
- Paper trading and live trading (v0.4 and v1.0)
- ML strategy infrastructure (v0.3)
- Dashboard or UI (v0.4)
- Anything involving options, futures, crypto, or non-US equities (deferred or permanent)

The hooks for these features must be in place (protocols and optional parameters) but the implementations are not v0.1 work.

    start: Timestamp
    end: Timestamp

        if self.start > self.end:
            raise ValueError(f"start {self.start} > end {self.end}")
    
    def days(self) -> int:
        return (self.end - self.start).days
    
    def contains(self, t: Timestamp) -> bool:

### 4.5 `core/data.py` — the most important file in v0.1

```python
from __future__ import annotations
from typing import Protocol, runtime_checkable
import pandas as pd
from .types import Timestamp
from .exceptions import LookAheadError
from .universe import Universe


@runtime_checkable
class DataProvider(Protocol):
    """Provides point-in-time data access."""
    name: str
    
    def get_panel(
        self,
        fields: list[str],
        start: Timestamp,
        end: Timestamp,
        universe: Universe | None = None,
    ) -> pd.DataFrame:
        """
        Panel data: multi-index (timestamp, asset_id) rows, fields as columns.
        Must respect point-in-time: data for timestamp t reflects what was 
        known at t, not data revised later.
        """
    
    def get_returns(
        self,
        start: Timestamp,
        end: Timestamp,
        universe: Universe,
        frequency: str = "daily",
    ) -> pd.DataFrame:
        """Returns matrix: index = Timestamp, columns = AssetId."""
    
    def get_factor_returns(
        self,
        factors: list[str],
        start: Timestamp,
        end: Timestamp,
        frequency: str = "daily",
    ) -> pd.DataFrame:
        """For providers that publish factor returns directly (French, OSAP)."""


class AsOfDataProvider:
    """
    Wraps any DataProvider with strict 'as-of' semantics. Any data 
    access beyond as_of raises LookAheadError. Framework's primary 
    defense against look-ahead bias.
    """
    
    def __init__(self, inner: DataProvider, as_of: Timestamp):
        self._inner = inner
        self._as_of = as_of
        self.name = f"{inner.name}@{as_of.date()}"
    
    def get_panel(self, fields, start, end, universe=None):
        if end > self._as_of:
            raise LookAheadError(
                f"Strategy requested data through {end} but as_of is "
                f"{self._as_of}. This indicates look-ahead bias."
            )
        return self._inner.get_panel(fields, start, end, universe)
    
    def get_returns(self, start, end, universe, frequency="daily"):
        if end > self._as_of:
            raise LookAheadError(...)
        return self._inner.get_returns(start, end, universe, frequency)
    
    def get_factor_returns(self, factors, start, end, frequency="daily"):
        if end > self._as_of:
            raise LookAheadError(...)
        return self._inner.get_factor_returns(factors, start, end, frequency)
    
    @property
    def as_of(self) -> Timestamp:
        return self._as_of
```

### 5.2 `storage/cache.py`

```python
import duckdb
from pathlib import Path
import pandas as pd

class CacheLayer:
    """DuckDB + Parquet cache for reproducibility-first data access."""
    
    def __init__(self, cache_dir: Path | None = None):
        self.cache_dir = cache_dir or Path.home() / ".alpha_decay_foundry/cache"
        self.cache_dir.mkdir(parents=True, exist_ok=True)
        self._conn = duckdb.connect(str(self.cache_dir / "metadata/downloads.duckdb"))
        self._init_schema()
    
    def store(
        self,
        source: str,
        dataset: str,
        data: pd.DataFrame,
        version: str | None = None,
    ) -> None: ...
    
    def load(
        self,
        source: str,
        dataset: str,
        version: str | None = None,
        filters: dict | None = None,
    ) -> pd.DataFrame: ...
    
    def exists(self, source: str, dataset: str, version: str | None = None) -> bool: ...
    
    def query(self, sql: str) -> pd.DataFrame:
        """Arbitrary DuckDB query against cached Parquet files."""
```

class LongShortQuintile:
    """Top 20% long, bottom 20% short, equal-weighted within quintiles."""

### 9.2 `engines/vectorbt_engine.py`

VectorbtBacktestEngine implementation:

1. Determine rebalance dates from `strategy.rebalance_frequency` and trading calendar
2. For each rebalance `t`, wrap data with `AsOfDataProvider(data, as_of=t)`, call `strategy.target_weights()`
3. Collect weights into daily-frequency DataFrame (forward-filled between rebalances)
4. Fetch full price panel
5. Build vectorbt Portfolio via `vbt.Portfolio.from_orders` with `size_type='targetpercent'`
6. v0.1: `fees=0.0`, `slippage=0.0` (research configuration)
7. Convert to `BacktestResult`

**Critical**: AsOf wrapping happens at each rebalance, not once. Strategy called with progressively-newer `as_of` values. This enforces walk-forward by construction.

## 12. Workflow: Issue → PR → Audit

Encoded in `CONTRIBUTING.md`, enforced by repository conventions.

### 12.1 Issue creation

Before any code:
1. Open GitHub issue: `[v0.1] <module>: <specific change>`
2. Body: scope, acceptance criteria (binary checks), estimated effort (2-8 hours), references to PRD sections
3. Label `v0.1` and area (`core`, `storage`, `data-providers`, etc.)

### 12.2 Implementation

1. Create branch `v0.1/<issue-number>-<short-description>`
2. Implement against acceptance criteria
3. Write tests as you go
4. Keep PR under 500 lines
5. Run locally before push: `ruff check`, `mypy --strict`, `pytest`

### 12.3 PR review

1. Open PR linking to issue (`Closes #N`)
2. PR template: summary, acceptance criteria checkmarks, manual testing
3. CI must pass before audit
4. Audit agent reviews automatically
5. Verdict: approve / block-with-changes / escalate
6. Address findings, re-request review
7. Merge when audit approves

### 12.4 Audit agent

Separate Claude Code session running against each PR with `docs/audit-prompt.md`. Checks:

1. Acceptance criteria from issue demonstrably met
2. Tests cover new code with numeric/binary assertions
3. `mypy --strict` passes
4. No new `# type: ignore` without explanatory comment
5. No new TODO without GitHub issue reference
6. Design matches PRD
7. No scope creep beyond stated scope
8. Documentation strings on all public methods
9. No new dependencies without justification
10. Grinold-Kahn references in analytics where appropriate

**Verdicts:**
- **Approve**: PR can merge
- **Block with changes**: specific changes requested
- **Escalate**: ambiguous architectural questions sent to Jacopo in plain English

**Escalation triggers:** any architectural deviation from PRD, any core protocol change, any new dependency, any disabled or weakened test.

## 14. Code conventions

- `from __future__ import annotations` at top of every file
- Type hints required everywhere; mypy --strict passes
- Google-style docstrings on public class and function
- Use framework's exception hierarchy
- `logging.getLogger(__name__)` per module; no print statements
- snake_case functions/variables, PascalCase classes, _leading_underscore private
- No lambdas in protocols or core code
- No side effects at module top-level
- Pandas: explicit `.loc[]` over chained indexing
- Datetime: always `pd.Timestamp` UTC-aware
- Grinold-Kahn analytics docstrings include chapter citation
